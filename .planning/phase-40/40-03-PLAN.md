  <action>
Refactor `src/lib/square/fetch-client.ts` to accept `SquareConfig` as the first parameter for every exported function. This is a mechanical refactoring -- every function signature changes but the body logic stays the same.

**Step 1: Replace module-level constants and helpers**

Remove these (lines 1-24):
```
const SQUARE_BASE_URL = process.env.SQUARE_ENVIRONMENT === 'production' ...
function getHeaders() { ... }
function getLocationId() { ... }
```

Replace with:
```typescript
import type { SquareConfig } from './types'

const SQUARE_VERSION = '2024-12-18'

type SquareRequestBody = Record<string, unknown>

function getBaseUrl(config: SquareConfig): string {
  return config.environment === 'production'
    ? 'https://connect.squareup.com'
    : 'https://connect.squareupsandbox.com'
}

function getHeaders(config: SquareConfig) {
  return {
    'Square-Version': SQUARE_VERSION,
    'Authorization': `Bearer ${config.accessToken}`,
    'Content-Type': 'application/json'
  }
}
```

**Step 2: Add `config: SquareConfig` as the FIRST parameter to every exported function:**

- `listCatalogObjects(config: SquareConfig, types?: string[], cursor?: string)`
- `searchCatalogItems(config: SquareConfig, query: SquareRequestBody)`
- `searchLocationCatalogItems(config: SquareConfig)`
- `searchAllCatalogItems(config: SquareConfig)`
- `getOrder(config: SquareConfig, orderId: string)`
- `createOrder(config: SquareConfig, orderData: SquareRequestBody)`
- `createPayment(config: SquareConfig, paymentData: SquareRequestBody)`
- `listLocations(config: SquareConfig)`
- `listCatalogTaxes(config: SquareConfig)`
- `createCatalogTax(config: SquareConfig, taxData: SquareRequestBody)`
- `upsertCatalogItem(config: SquareConfig, itemData: SquareRequestBody)`
- `upsertCatalogCategory(config: SquareConfig, categoryData: SquareRequestBody)`
- `deleteCatalogObject(config: SquareConfig, objectId: string)`
- `batchUpsertCatalogObjects(config: SquareConfig, objects: SquareRequestBody[])`

**Step 3: In each function body, replace:**
- `SQUARE_BASE_URL` with `getBaseUrl(config)`
- `getHeaders()` with `getHeaders(config)`
- `getLocationId()` with `config.locationId`

**Step 4: For `searchLocationCatalogItems`, update the body:**

After this refactoring, there should be ZERO references to `process.env` anywhere in the file.
