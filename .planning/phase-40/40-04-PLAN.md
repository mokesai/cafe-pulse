**orders.ts changes (most complex -- has module-level cache to fix):**

1. Add import: `import type { SquareConfig } from './types'`

2. **Replace the module-level singleton cache with a tenant-scoped Map:**

   Remove:
   ```typescript
   let catalogItemsCache: CatalogObject[] | null = null
   let cacheTimestamp = 0
   const CACHE_DURATION = 5 * 60 * 1000
   ```

   Replace with:
   ```typescript
   const CACHE_DURATION = 5 * 60 * 1000 // 5 minutes
   const catalogCacheByTenant = new Map<string, { items: CatalogObject[]; expiresAt: number }>()
   ```

3. **Update `getCatalogItems` to accept config + tenantId:**
   ```typescript
   async function getCatalogItems(config: SquareConfig, tenantId: string): Promise<CatalogObject[]> {
     const cached = catalogCacheByTenant.get(tenantId)
     if (cached && cached.expiresAt > Date.now()) {
       return cached.items
     }

     try {
       const result = await searchAllCatalogItems(config) as CatalogSearchResponse
       const items = result.objects?.filter((obj): obj is CatalogObject => obj.type === 'ITEM') || []

       catalogCacheByTenant.set(tenantId, { items, expiresAt: Date.now() + CACHE_DURATION })
       return items
     } catch (error) {
       console.error('Error fetching catalog items:', error)
       const stale = catalogCacheByTenant.get(tenantId)
       return stale?.items || []
     }
   }
   ```

4. **Update `getVariationIdForItem` to accept config + tenantId:**
   ```typescript
   async function getVariationIdForItem(config: SquareConfig, tenantId: string, itemId: string): Promise<string>
   ```
   Pass config and tenantId through to `getCatalogItems(config, tenantId)`.

5. **Update exported function signatures to add `config: SquareConfig` and `tenantId: string`:**
   - `previewSquareOrder(config: SquareConfig, tenantId: string, items: SimpleCartItem[])`
   - `createSquareOrder(config: SquareConfig, tenantId: string, items: SimpleCartItem[])`
   - `processPayment(config: SquareConfig, paymentToken: string, orderId: string | null, amount: number, customerEmail?: string)`

   Note: `processPayment` does NOT need tenantId because it doesn't use the catalog cache.

6. **Pass config through to all fetch-client calls:**

   - `createPayment(config, paymentData)` instead of `createPayment(paymentData)`

   - `searchAllCatalogItems(config)` instead of `searchAllCatalogItems()`
