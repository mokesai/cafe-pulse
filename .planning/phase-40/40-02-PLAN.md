<task type="auto">
  <name>Task 2: Create credential loading layer with Vault + env fallback</name>
  <files>src/lib/square/config.ts</files>
  <action>
Create `src/lib/square/config.ts` implementing the credential loading pattern from the research. This file must:

1. **Import dependencies:**
   - `createServiceClient` from `@/lib/supabase/server`
   - `DEFAULT_TENANT_ID` from `@/lib/tenant/types`
   - `SquareConfig` from `./types`

2. **Cache setup using globalThis pattern (matches existing tenant/cache.ts pattern):**
   - Declare global `__squareConfigCache: Map<string, { config: SquareConfig; expiresAt: number }> | undefined`
   - `CACHE_TTL_MS = 60 * 1000` (60 seconds)
   - Helper `getCache()` that lazily initializes the Map on globalThis

3. **`getTenantSquareConfig(tenantId: string): Promise<SquareConfig | null>`:**
   - Check cache first: if cached entry exists and `expiresAt > Date.now()`, return it
   - For default tenant (`tenantId === DEFAULT_TENANT_ID`), try env var fallback first via `getEnvSquareConfig()`. If env vars are set, cache and return
   - For ALL tenants (including default if env vars not set): call `createServiceClient().rpc('get_tenant_square_credentials_internal', { p_tenant_id: tenantId })`
   - If RPC returns data, map the snake_case fields to SquareConfig camelCase fields:
     - `data.access_token` -> `accessToken`
     - `data.application_id` -> `applicationId`
     - `data.location_id` -> `locationId`
     - `data.environment` -> `environment` (cast to 'sandbox' | 'production')
     - `data.merchant_id` -> `merchantId`
     - `data.webhook_signature_key` -> `webhookSignatureKey`
   - Cache the result and return
   - If RPC fails or returns no data, return null

4. **`getEnvSquareConfig(): SquareConfig | null` (private helper):**
   - Read `SQUARE_ACCESS_TOKEN`, `SQUARE_APPLICATION_ID`, `SQUARE_LOCATION_ID` from `process.env`
   - If any of the three required vars are missing, return null
   - Return SquareConfig with `environment` from `process.env.SQUARE_ENVIRONMENT || 'sandbox'`
   - `merchantId` from `process.env.SQUARE_MERCHANT_ID ?? undefined`
   - `webhookSignatureKey` from `process.env.SQUARE_WEBHOOK_SIGNATURE_KEY ?? undefined`

5. **Export only `getTenantSquareConfig`** (not the env helper or cache internals).

IMPORTANT: The RPC call returns an array (Supabase rpc for RETURNS TABLE). Access `data[0]` or handle accordingly. If `data` is empty array or null, return null.

Do NOT import from `next/headers` or `next/cache` in this file. It must work in any server context (API routes, server components, scripts).
  </action>
