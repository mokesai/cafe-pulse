<task type="auto">
  <name>Task 1: Create Vault credential migration</name>
  <files>supabase/migrations/20260215000000_vault_square_credentials.sql</files>
  <action>
Create a SQL migration that does ALL of the following in a single file:

1. **Add vault_secret_id columns to tenants table:**
   - `square_access_token_vault_id uuid REFERENCES vault.secrets(id)` (nullable)
   - `square_webhook_key_vault_id uuid REFERENCES vault.secrets(id)` (nullable)
   - Use `ADD COLUMN IF NOT EXISTS` for idempotency

2. **Create `get_tenant_square_credentials_internal(p_tenant_id uuid)` function:**
   - SECURITY DEFINER, `SET search_path = public`
   - RETURNS TABLE(access_token text, application_id text, location_id text, environment text, merchant_id text, webhook_signature_key text)
   - Reads tenant row, checks Vault first (`vault.decrypted_secrets`), falls back to plain columns (`square_access_token`, `square_webhook_signature_key`)
   - Returns empty if tenant not found or inactive
   - REVOKE ALL FROM PUBLIC, GRANT EXECUTE to service_role only

3. **Create `get_tenant_square_credentials(p_tenant_id uuid)` function (owner-facing):**
   - SECURITY DEFINER, `SET search_path = public`
   - Same return type as internal function
   - Checks `auth.uid()` is tenant owner via `tenant_memberships` before delegating to internal function
   - RAISE EXCEPTION if not authorized

4. **Create `set_tenant_square_credentials(p_tenant_id, p_access_token, p_webhook_signature_key)` function:**
   - SECURITY DEFINER, `SET search_path = public`
   - Checks `auth.uid()` is tenant owner
   - If vault entry exists for the credential, updates it via `vault.update_secret()`
   - If no vault entry, creates via `vault.create_secret()` and stores UUID on tenants row
   - Handle both access_token and webhook_signature_key independently (either can be NULL to skip)

5. **Create credential_audit_log table:**
   - `id uuid DEFAULT gen_random_uuid() PRIMARY KEY`
   - `tenant_id uuid REFERENCES public.tenants(id) NOT NULL`
   - `user_id uuid REFERENCES auth.users(id)` (nullable for service_role operations)
   - `action text NOT NULL CHECK (action IN ('create', 'update', 'delete'))`
   - `credential_type text NOT NULL`
   - `created_at timestamptz DEFAULT now()`
   - Enable RLS, no SELECT policies (service_role only reads audit logs)
   - Add `tenant_id` column with tenant_id for consistency with Phase 20 pattern

6. **Create merchant_id index:**
   - `CREATE INDEX IF NOT EXISTS idx_tenants_square_merchant_id ON public.tenants (square_merchant_id) WHERE square_merchant_id IS NOT NULL`

Use `CREATE OR REPLACE FUNCTION` for idempotency on all functions. Follow the exact SQL patterns from 40-RESEARCH.md.
  </action>
