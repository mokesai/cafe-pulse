<objective>
Refactor all 7 admin API routes that read Square env vars directly. These routes bypass fetch-client and have their own inline Square API calls (getHeaders(), SQUARE_BASE_URL, etc.). Each must resolve tenant context and use getTenantSquareConfig().

Purpose: These admin routes are the biggest source of env var duplication. Without this, admin inventory sync, push, sales, and menu management would all use the wrong tenant's credentials.
Output: Seven updated admin API routes with tenant-aware credential loading.
</objective>

  <files>src/app/api/admin/inventory/sync-square/route.ts, src/app/api/admin/inventory/push-to-square/route.ts, src/app/api/admin/inventory/square-search/route.ts</files>
  <action>
Each of these 3 routes has a similar pattern: module-level or function-level env var reads for Square credentials and inline Square API calls. Apply the same transformation to each:

**For each file:**

1. Add imports:
   ```typescript
   import { getCurrentTenantId } from '@/lib/tenant/context'
   import { getTenantSquareConfig } from '@/lib/square/config'
   import type { SquareConfig } from '@/lib/square/types'
   ```

2. Remove ALL module-level Square env var reads. For example in sync-square/route.ts, remove:
   ```typescript
   const squareAccessToken = process.env.SQUARE_ACCESS_TOKEN!
   const squareEnvironment = process.env.SQUARE_ENVIRONMENT
   const squareLocationId = process.env.SQUARE_LOCATION_ID!
   ```

3. Remove module-level derived constants like:
   ```typescript
   const SQUARE_BASE_URL = squareEnvironment === 'production' ? ... : ...
   ```

4. **For sync-square/route.ts specifically:** Also remove the module-level Supabase client singleton:
   ```typescript
   const supabase = createClient(supabaseUrl, supabaseServiceKey)
   ```
   Replace with per-request `createServiceClient()` from `@/lib/supabase/server` inside the handler.

5. At the top of each POST/GET handler, resolve tenant:
   ```typescript
   const tenantId = await getCurrentTenantId()
   const squareConfig = await getTenantSquareConfig(tenantId)
   if (!squareConfig) {
     return NextResponse.json({ error: 'Square not configured' }, { status: 503 })
   }
   ```

6. Replace all references to removed variables with SquareConfig properties:
   - `squareAccessToken` -> `squareConfig.accessToken`
   - `squareLocationId` -> `squareConfig.locationId`
   - `SQUARE_BASE_URL` -> derive per-call: `const baseUrl = squareConfig.environment === 'production' ? 'https://connect.squareup.com' : 'https://connect.squareupsandbox.com'`
   - Inline `getHeaders()` functions -> use `squareConfig.accessToken` directly in Authorization header

7. Any inline `getHeaders()` function should use the config:
   ```typescript
   const headers = {
     'Square-Version': '2024-12-18',
     'Authorization': `Bearer ${squareConfig.accessToken}`,
     'Content-Type': 'application/json'
   }
   ```

Keep all existing business logic (sync logic, error handling, admin auth checks) unchanged.
  </action>

  </verify>
  <done>Three inventory admin routes resolve tenant and use getTenantSquareConfig(). Zero env var reads for Square credentials.</done>
