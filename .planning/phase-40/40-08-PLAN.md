**SquareProvider.tsx changes -- extend the context to expose applicationId and locationId:**

The existing `SquareProvider` at `src/providers/SquareProvider.tsx` receives `applicationId` and `locationId` as props but only exposes `{ payments, isLoading, error }` via context. Extend the context to also include `applicationId` and `locationId` so descendants (like CheckoutModal) can access them without prop threading.

1. Update the `SquareContextType` interface:
   ```typescript
   interface SquareContextType {
     payments: SquarePayments | null
     isLoading: boolean
     error: string | null
     applicationId: string
     locationId: string
   }
   ```

2. Update the default context value:
   ```typescript
   const SquareContext = createContext<SquareContextType>({
     payments: null,
     isLoading: true,
     error: null,
     applicationId: '',
     locationId: ''
   })
   ```

3. Update the Provider value to include applicationId and locationId:
   ```typescript
   <SquareContext.Provider value={{ payments, isLoading, error, applicationId, locationId }}>
   ```

4. Add a second hook for config access (simpler than useSquarePayments for components that just need config):
   ```typescript
   export function useSquareConfig() {
     const context = useContext(SquareContext)
     return { applicationId: context.applicationId, locationId: context.locationId }
   }
   ```

**DynamicSquareProvider.tsx changes -- accept props instead of fetching:**

Refactor DynamicSquareProvider from fetching config internally to receiving it via props:

```typescript
'use client'

import { ReactNode } from 'react'
import { SquareProvider } from '@/providers/SquareProvider'

export interface SquarePublicConfig {
  applicationId: string
  locationId: string
  environment: 'sandbox' | 'production'
}

interface DynamicSquareProviderProps {
  children: ReactNode
  config: SquarePublicConfig | null
}

export default function DynamicSquareProvider({ children, config }: DynamicSquareProviderProps) {
  if (!config) {
    // No Square config = render children without payments SDK
    // This handles unconfigured tenants gracefully
    return <>{children}</>
  }

  return (
    <SquareProvider
      applicationId={config.applicationId}
      locationId={config.locationId}
      environment={config.environment}
    >
      {children}
    </SquareProvider>
  )
}
```

Key changes:
- Remove `useState`, `useEffect`, and the `fetch('/api/square/config')` call entirely
- Remove loading state (no longer needed -- config comes from server render)
- Remove error state handling (null config = no Square)
- Accept `config: SquarePublicConfig | null` as a prop
- If config is null, render children without SquareProvider wrapper
- Keep 'use client' directive (SquareProvider is a client component)
- Export the `SquarePublicConfig` interface
