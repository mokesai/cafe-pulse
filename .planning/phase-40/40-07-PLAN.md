<objective>
Update both Square webhook handlers to resolve tenant by `merchant_id` from the webhook payload, then use that tenant's credentials for signature verification and subsequent API calls.

Purpose: Webhooks don't have cookie-based tenant context. They must identify the tenant from the `merchant_id` field that Square includes in every webhook payload. This enables multi-tenant webhook processing through a single shared endpoint.
Output: Two updated webhook handlers with merchant_id-based tenant resolution.
</objective>

   Replace all `getSupabaseClient()` calls with `createServiceClient()` from `@/lib/supabase/server`. Remove the `getSupabaseClient()` function entirely.

6. **Update the POST handler flow:**
   ```typescript
   export async function POST(request: NextRequest) {
     try {
       const body = await request.text()
       const event: SquareCatalogWebhookEvent = JSON.parse(body)

       // Resolve tenant from merchant_id
       const tenantId = await resolveTenantFromMerchantId(event.merchant_id)
       if (!tenantId) {
         console.warn(`Unknown merchant_id in catalog webhook: ${event.merchant_id}`)
         return NextResponse.json({ success: false, message: 'Unknown merchant' }, { status: 200 })
       }

       // Load tenant's Square config
       const squareConfig = await getTenantSquareConfig(tenantId)
       if (!squareConfig) {
         console.warn(`No Square config for tenant ${tenantId}`)
         return NextResponse.json({ success: false, message: 'Tenant not configured' }, { status: 200 })
       }

       // Verify webhook signature using tenant's key
       const headersList = await headers()
       const signature = headersList.get('x-square-signature') || ''
       if (squareConfig.webhookSignatureKey && !verifySquareSignature(body, signature, squareConfig.webhookSignatureKey)) {
         console.error('Invalid webhook signature')
         return NextResponse.json({ error: 'Invalid signature' }, { status: 401 })
       }

       // ... rest of handler continues with squareConfig passed to API calls
     }
   }
   ```

7. **Update `fetchCatalogChanges` to accept SquareConfig:**
   ```typescript
   async function fetchCatalogChanges(config: SquareConfig, sinceTimestamp?: Date): Promise<CatalogResponse> {
     const baseUrl = config.environment === 'production'

     // ... use makeSquareHeaders(config) and baseUrl instead of getSquareHeaders() and SQUARE_BASE_URL
   ```
