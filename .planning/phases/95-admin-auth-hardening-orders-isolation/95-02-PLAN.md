<objective>
Migrate three admin inventory/dashboard routes from legacy auth patterns to requireAdminAuth(). These routes use either inline profiles.role checks (stats) or the validateAdminAccess(adminEmail) pattern that takes an admin email from the POST body. None of these routes have data query issues — only the auth layer needs updating.

Purpose: The validateAdminAccess pattern is the weakest auth in the codebase — anyone who knows an admin email address can call these endpoints. Migrating to requireAdminAuth() gives CSRF protection, rate limiting, and proper tenant_memberships-based role checks.

Output: Three routes updated to requireAdminAuth() with adminEmail removed from interfaces and request bodies.
</objective>
