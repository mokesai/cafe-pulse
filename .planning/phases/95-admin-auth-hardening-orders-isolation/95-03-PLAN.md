<objective>
Migrate bulk-upload and hybrid-sync from legacy email-based auth to requireAdminAuth(), add tenant_id to bulk-upload's INSERT payload, tenant-scope hybrid-sync's inventory queries, and eliminate hybrid-sync's internal HTTP self-call to sync-square by inlining the sync logic via shared functions.

Purpose: bulk-upload currently inserts inventory_items without tenant_id — a data integrity bug that would cause failures under NOT NULL constraints on a second tenant. hybrid-sync calls sync-square via HTTP, which will break after sync-square's auth migration (Plan 02) since the internal call has no auth cookies. The fix is to extract the sync-square core logic into a shared library and call it directly.

Depends on Plan 02 because hybrid-sync's direct call approach imports functions from sync-square's logic, and sync-square needs to be in its final state first (or the shared functions extracted before both are called).

Output: Two routes fully migrated with correct auth, data integrity, and no HTTP self-calls.
</objective>

hybrid-sync has the most complex migration: auth replacement, tenant scoping on data queries, AND elimination of the HTTP self-call to sync-square.

**The HTTP self-call problem:** `runSquareSync()` calls `fetch(${NEXT_PUBLIC_SITE_URL}/api/admin/inventory/sync-square, ...)` passing `adminEmail` in the body. After sync-square is migrated (Plan 02), this call will fail — it has no auth cookies and won't pass CSRF. The fix is to call the sync logic directly.

**Extract shared sync logic approach:**
The sync-square route contains these pure, reusable functions:
- `fetchSquareCatalog(config: SquareConfig)` — fetches Square catalog
- `getSupplierMappings(supabase)` — queries suppliers table
- `getExistingInventoryItems(supabase, tenantId)` — queries inventory_items
- `processSquareCatalog(catalogData, suppliers, existingSquareIds)` — pure computation
- `syncInventoryItems(supabase, tenantId, items, dryRun)` — inserts items

Since these functions are in a Next.js API route file (`route.ts`) they cannot be imported by other files (Next.js only allows exporting GET/POST/etc. from route.ts files, not utility functions for cross-import). Therefore, **do not try to import from sync-square/route.ts**.

Instead, inline the Square catalog sync logic directly into hybrid-sync's `runSquareSync` function, replacing the HTTP fetch pattern:

1. In `hybrid-sync/route.ts`, add imports for the Square config:
   ```typescript
   import { createServiceClient } from '@/lib/supabase/server'
   import { getCurrentTenantId } from '@/lib/tenant/context'
   import { getTenantSquareConfig } from '@/lib/square/config'
   import { requireAdminAuth, isAdminAuthSuccess } from '@/lib/admin/middleware'
   ```
   Remove `import { createClient } from '@supabase/supabase-js'`

2. Remove `getSupabaseClient()` function. Remove `validateAdminAccess()` function.

3. Replace `runSquareSync(adminEmail, dryRun)` with `runSquareSync(supabase, tenantId, squareConfig, dryRun)` that contains the sync logic inline (fetching catalog from Square, getting suppliers, getting existing items, processing, inserting). The logic mirrors what sync-square/route.ts does in its POST handler. Keep it simple — the inline version can be a condensed version:

   ```typescript
   async function runSquareSync(
     supabase: ReturnType<typeof createServiceClient>,
     tenantId: string,
     squareConfig: SquareConfig,
     dryRun: boolean
   ) {
     try {
       const SQUARE_VERSION = '2024-12-18'
       const baseUrl = squareConfig.environment === 'production'
         ? 'https://connect.squareup.com'
         : 'https://connect.squareupsandbox.com'

       const catalogResponse = await fetch(`${baseUrl}/v2/catalog/search`, {
         method: 'POST',
         headers: {
           'Square-Version': SQUARE_VERSION,
           'Authorization': `Bearer ${squareConfig.accessToken}`,
           'Content-Type': 'application/json'
         },
         body: JSON.stringify({ object_types: ['ITEM', 'CATEGORY'], include_related_objects: true })
       })
       if (!catalogResponse.ok) {
         throw new Error(`Square API error: ${catalogResponse.status}`)
       }
       const catalogData = await catalogResponse.json()

       // Get existing inventory to avoid duplicates
       const { data: existingItems } = await supabase
         .from('inventory_items')
         .select('square_item_id')
         .eq('tenant_id', tenantId)
       const existingIds = new Set((existingItems ?? []).map(i => i.square_item_id))

       // Identify new items from catalog
       const objects = catalogData.objects || []
       const newItems = objects
         .filter((obj: { type: string; id: string }) => obj.type === 'ITEM' && !existingIds.has(obj.id))
         .map((obj: { id: string; item_data?: { name?: string } }) => ({
           square_item_id: obj.id,
           item_name: obj.item_data?.name || 'Unknown Item',
           current_stock: 0,
           minimum_threshold: 5,
           reorder_point: 10,
           unit_cost: 0,
           unit_type: 'each' as const,
           is_ingredient: true,
           location: 'main',
           notes: 'Synced from Square catalog via hybrid sync',
           tenant_id: tenantId,
         }))

       if (!dryRun && newItems.length > 0) {
         const { error } = await supabase.from('inventory_items').insert(newItems)
         if (error) throw new Error(`Failed to insert items: ${error.message}`)
       }

       return { success: true, summary: { newItems: newItems.length, dryRun } }
     } catch (error) {
       return { success: false, error: error instanceof Error ? error.message : 'Unknown error' }
     }
   }
   ```
