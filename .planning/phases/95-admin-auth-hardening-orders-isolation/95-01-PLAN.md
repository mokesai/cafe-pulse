<objective>
Migrate admin/orders API route from the pre-Phase-50 `profiles.role` auth pattern to `requireAdminAuth()`, and add the missing tenant_id filters on the GET count query and PATCH UPDATE query.

Purpose: The PATCH handler uses createServiceClient() (bypasses RLS) and has no .eq('tenant_id', tenantId) on the UPDATE — this is an exploitable cross-tenant write gap. The GET count query omits the tenant filter so pagination totals include all tenants' orders.

Output: src/app/api/admin/orders/route.ts with requireAdminAuth() on both handlers, .eq('tenant_id', tenantId) on the count query, and .eq('tenant_id', tenantId) on the PATCH UPDATE.
</objective>
