<objective>
Add tenant_id filtering to the 15 COGS admin routes and 17 inventory admin routes identified as FAIL in the 70-02 security audit. Without these fixes, an admin user for Tenant A can read or modify Tenant B's COGS and inventory data.

Purpose: Admin routes sit behind requireAdminAuth() which validates the user's role, but they do not restrict which tenant's data is returned. A super-admin or future multi-tenant deployment would expose all tenants' data through these endpoints.

Output: 32 admin route files updated with consistent tenant filtering. The established pattern (from invoices/route.ts and other PASS routes) is followed exactly.
</objective>

  <action>
    Apply the same pattern used in src/app/api/admin/invoices/route.ts (which already PASSES the audit) to all 15 COGS routes.

    The standard pattern is:
    ```typescript
    import { getCurrentTenantId } from '@/lib/tenant/context'

    // Inside each handler (GET, POST, PUT, DELETE, PATCH):
    const tenantId = await getCurrentTenantId()

    // On every .from() query:
    .eq('tenant_id', tenantId)

    // On every INSERT:
    { tenant_id: tenantId, ...otherFields }
    ```

    For each of the 15 COGS route files:
    1. Add the import for getCurrentTenantId if not already present:
       `import { getCurrentTenantId } from '@/lib/tenant/context'`
    2. In each exported handler function (GET, POST, PUT, DELETE, PATCH): add `const tenantId = await getCurrentTenantId()` immediately after the auth check passes (after the requireAdminAuth check or after auth validation).
    3. Add `.eq('tenant_id', tenantId)` to every SELECT/UPDATE/DELETE query that does NOT already have it.
    4. Add `tenant_id: tenantId` to every INSERT payload.

    Per-file details from the audit:

    cogs/catalog/sync-square/route.ts: queries cogs_products (lines 204, 214), cogs_sellables (line 243). Also add tenant_id to any INSERT/UPSERT operations in the sync logic.

    cogs/modifier-option-recipes/[id]/route.ts: queries cogs_modifier_option_recipes (lines 47, 119, 129), cogs_modifier_option_recipe_lines (lines 57, 144). Filter all reads by tenant_id; include tenant_id in inserts.

    cogs/modifier-option-recipes/route.ts: queries cogs_modifier_option_recipes (lines 41, 103), cogs_modifier_option_recipe_lines (line 119).

    cogs/modifiers/options/route.ts: queries cogs_modifier_options (lines 19, 58).

    cogs/modifiers/seen/route.ts: queries sales_transaction_items (line 30).

    cogs/modifiers/sets/route.ts: queries cogs_modifier_sets (lines 18, 52).

    cogs/periods/[id]/close/route.ts: queries invoices (line 27), inventory_items (line 56), inventory_valuations (line 82), cogs_periods (line 97), cogs_reports (line 109).

    cogs/periods/[id]/export/route.ts: queries cogs_periods (line 34), cogs_reports (line 44), inventory_valuations (line 54).

    cogs/periods/route.ts: queries cogs_periods (lines 24, 64). Already has requireAdminAuth — add tenantId after auth check.

    cogs/product-recipes/[id]/route.ts: queries cogs_product_recipes (lines 47, 129, 139), cogs_product_recipe_lines (lines 57, 156).

    cogs/sellable-overrides/route.ts: queries cogs_sellable_recipe_overrides (lines 117, 156), cogs_sellable_recipe_override_ops (line 172).

    cogs/sellables/route.ts: multiple queries (read the file to find them all).

    Important rules:
    - Do NOT add tenant_id filtering to auth.users queries or supabase.auth.* calls — those are always global.
    - Do NOT add tenant_id to queries on the tenants table itself.
    - Do add tenant_id to count queries (.select('*', { count: 'exact', head: true })) as well.
    - For UPDATE and DELETE with .eq('id', someId), also add .eq('tenant_id', tenantId) as an additional safety filter.
