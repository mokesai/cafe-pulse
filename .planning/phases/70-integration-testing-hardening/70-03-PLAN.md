<objective>
Fix cross-tenant localStorage pollution by prefixing all localStorage keys with tenant slug or ID. Currently, cart data uses hardcoded keys ('cafe-cart', 'cafe-selected-variations') that are shared across all tenants on the same domain, causing Tenant A to see Tenant B's cart when switching tenants.

Purpose: Prevent localStorage leakage between tenants. Browser localStorage is domain-scoped, not subdomain-scoped on localhost, so tenant-a.localhost and tenant-b.localhost share the same localStorage. Keys must be prefixed with tenant identifier.

Output: Tenant-aware localStorage utility module, updated cart hooks using prefixed keys, verified isolation via manual testing or E2E tests from Plan 70-01.
</objective>
