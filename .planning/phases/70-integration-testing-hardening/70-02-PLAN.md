<objective>
Audit all service-role queries and module-level caches to identify potential cross-tenant data leakage risks. Service-role clients bypass RLS policies, so explicit tenant_id filtering is critical. Module-level caches persist across requests, requiring tenant-scoped keys.

Purpose: Ensure every createServiceClient() query explicitly filters by tenant_id (except platform admin routes). Verify all globalThis caches use tenant-scoped keys to prevent Tenant A seeing cached data from Tenant B.

Output: Two audit scripts and a comprehensive AUDIT_RESULTS.md report documenting all service-role queries and caches, with pass/warning/fail status for each usage.
</objective>
