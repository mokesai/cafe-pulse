### Root Cause Analysis

**Original Error (from screenshot):**
```
Runtime Error - Server
Failed to load tenant identity: column tenants.logo_url does not exist

src/lib/tenant/identity.ts (39:11) @ eval
```

**Root Cause:**
- Phase 50-01 added branding column SELECTs to getTenantIdentity() function
- But the actual database columns didn't exist until Phase 50-06 migration
- Both site and admin layouts call getTenantIdentity() on every page load
- Database error crashed the layout before OrdersManagement component could render
- This was misdiagnosed as a webpack bundling issue in initial investigation

**Fix Applied:**
- Phase 50-06: Migration `20260215140000_add_tenant_branding_columns.sql` added logo_url, primary_color, secondary_color columns
- Phase 50.1-01: Re-enabled OrdersManagement component (removed maintenance placeholder)

**Verification:**
- Migration file exists and adds required columns
- getTenantIdentity() selects those columns (line 31)
- Layouts call getTenantIdentity() (site line 32, admin line 15)
- Orders page imports and renders OrdersManagement
- No database column errors should occur
