    7. Create helper function `getStatusBadgeVariant(status: TenantStatus)` that returns:
       - 'default' for trial (blue)
       - 'success' for active (green)
       - 'warning' for paused (yellow)
       - 'destructive' for suspended (red)
       - 'secondary' for deleted (gray)

    8. Create async page function with searchParams: `{ searchParams: { q?: string, sort?: string } }` that:
       - Calls `await requirePlatformAdmin()`
       - Gets service client
       - Queries tenants with filters:
         ```typescript
         let query = supabase
           .from('tenants')
           .select('id, slug, name, status, created_at, trial_expires_at')
           .is('deleted_at', null);

         // Search filter
         if (searchParams.q) {
           query = query.or(`slug.ilike.%${searchParams.q}%,name.ilike.%${searchParams.q}%`);
         }

         // Sort
         const sortField = searchParams.sort === 'status' ? 'status' : 'created_at';
         query = query.order(sortField, { ascending: false });

         const { data: tenants, error } = await query;
         ```

    9. Render page with:
       - Page title "All Tenants" with count
       - Search input (client-side form with input[name="q"] that submits to ?q=...)
       - Sort dropdown (Created Date / Status)
       - Table with columns: Slug, Name, Status, Created, Trial Expires, Actions
       - Each row:
         * Slug as link to /platform/tenants/[id]
         * Name in bold
         * Status as Badge using getStatusBadgeVariant()
         * Created date formatted (new Date().toLocaleDateString())
         * Trial expires (if status === 'trial', show trial_expires_at or 'N/A')
         * Actions: "View" link to detail page
       - "Onboard New Tenant" button linking to /platform/tenants/new
       - Empty state if no tenants match search

    10. Use responsive table (horizontal scroll on mobile)
