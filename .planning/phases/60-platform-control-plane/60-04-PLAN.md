    Create src/app/api/platform/square-oauth/callback/route.ts:

    1. Mark as GET handler: `export async function GET(request: Request)`
    2. Import NextResponse from 'next/server'
    3. Import createServiceClient from '@/lib/supabase/server'
    4. Import parseOAuthState from '@/lib/square/config'

    5. Handler logic:
       - Parse query params: code, state, error
       - If error param present, redirect to /platform/tenants/new?error={error}
       - Parse state: `const { tenantId, environment } = parseOAuthState(state)`
       - Validate parsed state (if null, return error)
       - TODO: Verify state matches stored value (skip for now, add in hardening phase)

       - Determine token endpoint:
         * sandbox: 'https://connect.squareupsandbox.com/oauth2/token'
         * production: 'https://connect.squareup.com/oauth2/token'

       - Exchange authorization code for tokens:
         ```typescript
         const tokenResponse = await fetch(tokenEndpoint, {
           method: 'POST',
           headers: { 'Content-Type': 'application/json' },
           body: JSON.stringify({
             client_id: process.env.SQUARE_APPLICATION_ID,
             client_secret: process.env.SQUARE_SECRET,
             code,
             grant_type: 'authorization_code',
           }),
         });

         if (!tokenResponse.ok) {
           const error = await tokenResponse.text();
           console.error('Square token exchange failed:', error);
           return NextResponse.redirect(new URL('/platform/tenants/new?error=token_exchange_failed', request.url));
         }

         const tokens = await tokenResponse.json();
         // tokens = { access_token, refresh_token, expires_at, merchant_id, ... }
         ```

       - Store credentials in Vault:
         ```typescript
         const supabase = createServiceClient();
         await supabase.rpc('store_square_credentials_internal', {
           p_tenant_id: tenantId,
           p_environment: environment,
           p_access_token: tokens.access_token,
           p_refresh_token: tokens.refresh_token,
           p_merchant_id: tokens.merchant_id,
           p_expires_at: tokens.expires_at,
         });
         ```

       - Redirect to tenant detail page: `/platform/tenants/${tenantId}?success=square_connected`

    6. Add comprehensive error handling (missing env vars, network errors, RPC failures)
    7. Log errors but don't expose sensitive details to user

    Use pattern from 60-RESEARCH.md Pattern 2 (Square OAuth Code Flow - callback endpoint).
    Service client used to bypass RLS for Vault storage.
    expires_at from Square is ISO 8601 timestamp.
