<task type="auto">
  <name>Create platform_admins table</name>
  <files>supabase/migrations/20260216000001_create_platform_admins_table.sql</files>
  <action>
    Create migration that:
    1. Creates `platform_admins` table with columns:
       - id (UUID PRIMARY KEY DEFAULT gen_random_uuid())
       - user_id (UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE)
       - created_at (TIMESTAMPTZ NOT NULL DEFAULT NOW())
       - created_by (UUID REFERENCES auth.users(id)) - tracks who granted platform admin access
    2. Creates UNIQUE constraint on user_id (one platform_admins row per user)
    3. Creates btree index on user_id for fast lookups
    4. Enables RLS on platform_admins table
    5. Creates RLS policy "Platform admins can read all platform admins" FOR SELECT USING (EXISTS (SELECT 1 FROM platform_admins WHERE user_id = auth.uid()))
    6. Creates RLS policy "Only postgres can insert platform admins" FOR INSERT USING (false) - platform admin creation is manual/scripted only
    7. Adds table comment: 'Platform super-admins who can manage all tenants'
    8. Creates bootstrap function `bootstrap_platform_admin(admin_email TEXT)` that:
       - Checks if platform_admins table is empty
       - If empty, inserts user with matching email from auth.users
       - Returns success/failure message
       - Is SECURITY DEFINER to bypass RLS
       - Can only run if zero platform_admins exist (bootstrap-only)

    Reference patterns from existing tables (tenants, tenant_memberships).
    Migration must be idempotent.
  </action>
  <verify>
    ```bash

    psql $DATABASE_URL -c "SELECT policyname, cmd FROM pg_policies WHERE tablename = 'platform_admins';"

    # Verify bootstrap function exists

    ```
  </verify>
