## Detailed Findings

### Finding 1: Admin Orders PATCH — Missing Tenant Scope on UPDATE

**File:** `/Users/jerrym/Documents/projects/KP3-cafe/cafe-web/website/src/app/api/admin/orders/route.ts`
**Line:** 189-197
**Severity:** HIGH (data modification without tenant scope)

```typescript
const { data: updatedOrder, error: updateError } = await supabase
  .from('orders')
  .update(updates)
  .eq('id', orderId)  // No .eq('tenant_id', tenantId) here
  .select(...)
  .single()
```

The update uses `createServiceClient()` which bypasses RLS, and only filters by `orderId`. An admin of Tenant A could send a Tenant B `orderId` and update that order's status. The auth check also uses `profiles.role === 'admin'` (line 159-166) rather than tenant_memberships.

**Fix:** Add `.eq('tenant_id', tenantId)` to the UPDATE query and switch auth to `requireAdminAuth()`.

### Finding 2: Admin Orders GET — Pagination Count Missing Tenant Scope

**File:** `/Users/jerrym/Documents/projects/KP3-cafe/cafe-web/website/src/app/api/admin/orders/route.ts`
**Lines:** 106-124
**Severity:** MEDIUM (information leakage — counts from other tenants visible)

```typescript
let countQuery = supabase
  .from('orders')
  .select('*', { count: 'exact', head: true })
// No .eq('tenant_id', tenantId) applied to count query
```

Pagination shows the total count across ALL tenants, not just the current tenant. Orders themselves are correctly filtered, but the total count is wrong — which affects the "hasMore" pagination calculation.

**Fix:** Add `.eq('tenant_id', tenantId)` to the count query.

### Finding 3: Six Admin Routes Using Pre-Phase-50 Auth Pattern

**Files:**
- `/src/app/api/admin/orders/route.ts` (GET + PATCH)
- `/src/app/api/admin/dashboard/stats/route.ts`
- `/src/app/api/admin/inventory/hybrid-sync/route.ts`
- `/src/app/api/admin/inventory/bulk-upload/route.ts`
- `/src/app/api/admin/inventory/push-to-square/route.ts`
- `/src/app/api/admin/inventory/sync-square/route.ts`

**Severity:** MEDIUM (auth not tenant-scoped; data queries do have tenant_id filters)

These routes check `profiles.role === 'admin'` — a global role that predates multi-tenancy. This means a user with `profiles.role = 'admin'` on any tenant can call these endpoints, even if they are not a member of the current request's tenant. For the inventory routes, data queries do include `getCurrentTenantId()` + `.eq('tenant_id', tenantId)`, so data isolation holds. For orders PATCH, it does not (Finding 1 above).

**Fix:** Replace `profiles.role` check with `requireAdminAuth()` from `src/lib/admin/middleware.ts`.

### Finding 4: Soft-Deleted Tenants Remain Resolvable via Subdomain

**File:** `/Users/jerrym/Documents/projects/KP3-cafe/cafe-web/website/src/lib/tenant/context.ts`
**Line:** 29
**Severity:** LOW (defense-in-depth gap; data access still blocked by RLS)

`resolveTenantBySlug()` filters by `.eq('is_active', true)` but the `deleteTenant()` Server Action only sets `deleted_at` and `status='deleted'` — it does NOT set `is_active = false`. Because `resolveTenantBySlug()` uses `createServiceClient()` (bypasses RLS), a soft-deleted tenant is still resolvable via subdomain and will have its `x-tenant-id` cookie set.

The RLS policy on tenants table (`deleted_at IS NULL`) would prevent most data access, but the tenant resolution itself succeeds — potentially confusing.

**Fix (low priority):** Either add `is_active = false` to `deleteTenant()`, or add `.is('deleted_at', null)` to `resolveTenantBySlug()`.

### Finding 5: SQUARE_SECRET Env Var Undocumented

**File:** `/Users/jerrym/Documents/projects/KP3-cafe/cafe-web/website/src/app/api/platform/square-oauth/callback/route.ts`
**Lines:** 67-68, 93
**Severity:** LOW (operational gap)

The Square OAuth callback requires `process.env.SQUARE_SECRET` (the OAuth application secret, separate from the access token). This env var is not listed in CLAUDE.md's "Required `.env.local` variables" section. It would need to be set for the OAuth flow to work in any environment.

**Fix:** Add `SQUARE_SECRET` to the documented environment variables in CLAUDE.md and any env.example file.
